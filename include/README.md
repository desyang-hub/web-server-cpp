### 问题记录


- 1. 线程池退出前，需要确保任务都被完成
- 2. 对于容器的pop操作，使用一个变量来赋值接收，会触发不必要的拷贝构造，要是遇到一些特殊场景，比如变量不允许赋值时，直接就异常了
- 3. 任务队列中，框架无法判定用户任务是否可能异常，所以需要做必要的异常捕获，否则任务线程将直接结束
- 4. 线程池对象需要禁用拷贝构造和赋值构造
- 5. 对于vector，当得知容量时，可以提前进行预分配空间



epoll触发问题

1. 使用EPOLLIN触发+线程池的方式会有互斥问题
2. 使用EPOLLIN + EPOLLET + EPOLLONESHOT 方式，可以确保请求只会触发一次，除非触发之后，你自己通过EPOLL_CTL_MOD操作将监听继续添加回去，这样可以实现天然互斥
